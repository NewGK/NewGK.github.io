内部类及枚举类学习

​ 作者：NewGK

内部类的分类：

  • 成员内部类

  • 静态成员内部类

  • 局部内部类

  • 匿名内部类

    成员内部类：

    顾名思义，平时我们定义类中成员是普通的变量，现在我们可以定义成一个类

    静态成员内部类：

    与成员内部类不同的是，此类被 static 修饰，由于此原因，静态成员内部类可以直接由内部类类名访问

    局部内部类：

    局部内部类可以定义在外部类的各个作用域中，需要注意：局部内部类不可以被访问修饰符修饰

    匿名内部类：

    对接口的具体实现只实例化一次，或者继承父类的子类只需要实例化一次，可以使用匿名内部类，比如说

    线程的创建：

内部类访问外部类：

外部类.this.XXX

外部类访问内部类：

  • 外部类访问内部类时，需要先实例化内部类：

  • 当主类需要访问对象的内部类对象时，需要先实例化外部类再实例化内部类

使用内部类的好处：

好处有两点：

  • 内部类可以成为多继承的一种实现方式；
  • 可以将有一定逻辑关系的类组织到一起，又可以对外界隐藏。

枚举类：

枚举类是 JDK 1.5 的新特性，用关键字 “enum” 来声明，功能虽小，非常实用。

枚举类命名规范：

  • 类名以 “Enum” 为后缀

  • 成员一律大写，中间用下划线隔开

扩展枚举类：

可以再枚举类中写一些方法扩展枚举类的使用

枚举类应用场景：

  • 可以作为高级常量

  • 可以方便switch的判断
枚举类在 JVM 中是如何实现的？

  • 通过 “static final” 实现的，因此调用时才可以通过类名.成员来实现。

  • 也正因如此，枚举类不可继承、线程安全！！

枚举是否可以被序列化？

可以，枚举的序列化和普通类的序列化不同，枚举序列化的时候，只是将枚举对象的 name 属性输出到结果中，反序列化的时候则是通过 java.lang.Enum 的 valueOf 方法根据名字查找枚举对象
